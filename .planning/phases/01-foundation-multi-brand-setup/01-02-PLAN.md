    - hasPermission("manager", "brands", "delete") returns false
    - hasPermission("cashier", "menu", "read") returns true
    - requirePermission throws for unauthorized access
  </behavior>
  <action>
    **Types** (src/types/auth.ts):
    - SessionPayload: { userId, role, brandId, locationId, terminalId? }
    - LoginRequest: { email, password }
    - PinLoginRequest: { pin, terminalId }

    **Auth library** (src/lib/auth.ts):
    - Use jose (NOT jsonwebtoken — CommonJS issues with Edge runtime)
    - createSession(payload: SessionPayload): Promise<string> — signs JWT with HS256, 12h expiry (shift-length)
    - verifySession(token: string): Promise<SessionPayload> — verifies and returns payload
    - Use process.env.AUTH_SECRET encoded with TextEncoder

    **RBAC** (src/lib/permissions.ts):
    - Static permission map for 3 roles (admin, manager, cashier) — do NOT store in database
    - Resources: brands, locations, staff, menu, settings, reports, orders, sync
    - Actions: read, create, update, delete, sync
    - Admin: full access to all resources
    - Manager: read locations (own), read staff (own), read menu, read reports (own)
    - Cashier: read menu only
    - Export hasPermission(role, resource, action): boolean
    - Export requirePermission(role, resource, action): void (throws on denied)

    **Tests** (src/lib/__tests__/permissions.test.ts):
    - Test all permission combinations for the three roles
    - Test requirePermission throws correctly
    - Test edge cases (unknown resource returns false)
  </action>

<task type="auto">
  <name>Task 2: Implement auth API routes and route protection proxy</name>
  <files>
    src/app/api/auth/login/route.ts, src/app/api/auth/pin-login/route.ts,

  <action>
    **API Routes**:
    - POST /api/auth/login: Accepts {email, password}, validates against users table using bcryptjs, returns JWT in httpOnly cookie named "session" with 12h maxAge, SameSite=Lax, Secure in production. Returns 401 with {error: "Invalid credentials"} on failure.
    - POST /api/auth/pin-login: Accepts {pin, terminalId}, looks up terminal to get brandId+locationId, finds active users for that location, compares PIN hash using bcryptjs. Returns JWT in httpOnly cookie. Returns 401 on failure. Per user decision: terminal is pre-assigned to brand+location, staff just enter PIN.
    - GET /api/auth/verify: Reads session cookie, verifies JWT, returns {user: SessionPayload} or 401.

    **Route protection** (src/proxy.ts):
    - IMPORTANT: Next.js 16 uses proxy.ts with `export async function proxy()`, NOT middleware.ts
    - Public routes: /login, /admin/login, /api/auth/*
    - Admin routes (/admin/*): require session with role admin or manager
    - POS routes (everything else): require any valid session
    - Redirect unauthenticated to /login (POS) or /admin/login (admin)
    - Redirect cashier trying to access /admin back to /menu
  </action>
