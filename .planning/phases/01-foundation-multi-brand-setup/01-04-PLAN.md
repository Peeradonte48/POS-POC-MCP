  <action>
    **PIN Login Page** (src/app/(pos)/login/page.tsx):
    - Per user decision: Terminals are pre-assigned to a brand+location by admin — staff just see brand name, location, and PIN entry on login screen
    - Display terminal's brand name and location name at the top (fetched from terminal config or passed via query param)
    - Center the PinPad component on screen
    - On PIN submit: POST to /api/auth/pin-login with {pin, terminalId}
    - On success: redirect to /menu
    - On error: show error message (sonner toast), clear PIN
    - Per user decision: Quick user switching — show "Switch User" button at bottom of menu page that returns to PIN login without full logout. Terminal session stays active.
    - Clean, minimal design — brand logo, location name, PIN pad, that's it

    **PIN Pad** (src/components/pos/pin-pad.tsx):
    - 3x4 grid of large touch-friendly buttons (1-9, clear, 0, backspace)
    - PIN display shows dots (not digits) for security
    - Support 4-6 digit PINs (per user decision)
    - Auto-submit disabled — user taps Enter button to submit
    - Large button size (min 64x64px) for tablet touch targets
    - Disabled state while loading (prevent double-submit)
    - Use shadcn/ui Button component

    **Menu API** (src/app/api/menu/route.ts):
    - GET: Returns categories with nested items for the session's brandId
    - Query: categories with items, scoped by brandId from session
    - Include modifier groups and options nested under each item
    - Support ?categoryId query param to filter items by category
    - CRITICAL: Always filter by session.brandId to prevent cross-brand data leaks (per research pitfall #3)

    **Category Items API** (src/app/api/menu/[categoryId]/route.ts):
    - GET: Returns items for a specific category, brand-scoped
    - Include modifier groups and options
    - IMPORTANT: await params (Next.js 16 async params)

    **Hooks**:
    - useSession (src/hooks/use-session.ts): Fetches /api/auth/verify, returns { user, isLoading, isAuthenticated }. Uses @tanstack/react-query.
    - useTerminal (src/hooks/use-terminal.ts): Returns terminal context (brandId, locationId, brandName, locationName) from session.
    - useMenu (src/hooks/use-menu.ts): Fetches menu categories and items from /api/menu. Returns { categories, items, isLoading }. Accepts optional categoryId filter. Uses @tanstack/react-query with staleTime of 5 minutes (menu doesn't change often during a shift).
