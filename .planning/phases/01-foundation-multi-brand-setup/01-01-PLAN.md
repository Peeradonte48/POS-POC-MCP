  <action>
    Create the full Drizzle schema. Tables needed:

    **brands** (src/db/schema/brands.ts):
    - id (uuid PK), name, logoUrl, address, taxId, serviceChargePct (decimal default 10.00), vatPct (decimal default 7.00), createdAt, updatedAt

    **locations** (src/db/schema/locations.ts):
    - id (uuid PK), brandId (FK brands), name, address, settings (jsonb — printer config, table count, etc.), isActive, createdAt, updatedAt

    **users** (src/db/schema/users.ts):
    - id (uuid PK), brandId (FK brands), locationId (FK locations, nullable for admin), name, email (unique, nullable for PIN-only users), role (pgEnum: admin/manager/cashier), pinHash (nullable), passwordHash (nullable), isActive, createdAt, updatedAt
    - Constraint: at least one of pinHash or passwordHash must be set (enforce in app logic)

    **terminals** (src/db/schema/terminals.ts):
    - id (uuid PK), brandId (FK brands), locationId (FK locations), name, isActive, createdAt

    **menu_categories** (src/db/schema/menu.ts):
    - id (uuid PK), brandId (FK brands), parentId (uuid nullable self-ref for subcategories), name, sortOrder (int default 0), imageUrl (nullable), isActive (bool default true), erpId (varchar nullable)

    **menu_items** (src/db/schema/menu.ts):
    - id (uuid PK), brandId (FK brands), categoryId (FK menu_categories), name, description (text nullable), price (decimal 10,2), imageUrl (nullable), isActive (bool default true), erpId (varchar nullable), createdAt, updatedAt

    **modifier_groups** (src/db/schema/menu.ts):
    - id (uuid PK), menuItemId (FK menu_items), name, modifierType (pgEnum: single_select/multi_select), isRequired (bool default false), minSelections (int default 0), maxSelections (int nullable), sortOrder (int default 0)

    **modifier_options** (src/db/schema/menu.ts):
    - id (uuid PK), modifierGroupId (FK modifier_groups), name, priceAdjustment (decimal 10,2 default 0), isDefault (bool default false), isActive (bool default true), sortOrder (int default 0)

    **sync_logs** (src/db/schema/sync-logs.ts):
    - id (uuid PK), brandId (FK brands), status (varchar: success/error/in_progress), itemsSynced (int nullable), errorMessage (text nullable), startedAt (timestamp), completedAt (timestamp nullable)

    **Relations** (src/db/relations.ts):
    Define all Drizzle relations using the stable v0.45.x API (NOT defineRelations from v1 beta).

    **DB connection** (src/db/index.ts):
    Create pool with DATABASE_URL, export `db = drizzle(pool, { schema })`.

    **Seed** (src/db/seed.ts):
    Create a seed script that:
    - Creates 2 brands: "A RAMEN" (Japanese ramen) and "Burger Lab" (burgers)
    - Creates 1 location per brand
    - Creates 1 admin user (email: admin@aramen.com, password: admin123)
    - Creates 1 manager per location (PIN: 1234)
    - Creates 2 cashiers per location (PINs: 1111, 2222)
    - Creates menu categories and items for each brand (5 categories, 15 items each with modifiers)
    - Creates 1 terminal per location
    - Hash PINs and passwords with bcryptjs

    **Wave 0 Tests**: Create test stubs in __tests__ directories. Tests should validate schema shape (e.g., can construct insert objects with correct types). Use Vitest with in-memory assertions against the schema types — do NOT require a running database for unit tests.
