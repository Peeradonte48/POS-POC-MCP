```typescript
// Daily order number (in transaction, Bangkok timezone):
const today = new Date().toLocaleString('en-CA', { timeZone: 'Asia/Bangkok' }).split(',')[0];
// Or: sql`DATE(created_at AT TIME ZONE 'Asia/Bangkok') = ${today}`
const [{ count }] = await tx.select({ count: sql<number>`count(*)` })
  .from(orders)
  .where(and(eq(orders.locationId, session.locationId), sql`DATE(opened_at AT TIME ZONE 'Asia/Bangkok') = ${today}`));
const orderNumber = Number(count) + 1;

// Round number for add-items:
const [{ maxRound }] = await tx.select({ maxRound: sql<number>`MAX(round_number)` })
  .from(orderItems).where(eq(orderItems.orderId, orderId));
const nextRound = (maxRound ?? 0) + 1;
```

    - POST wraps in db.transaction: computes daily orderNumber (COUNT scoped to locationId + Bangkok date), inserts into orders, inserts each item into order_items with sentAt=new Date() and roundNumber=1, inserts modifiers into order_item_modifiers
    - POST returns 201 { orderId, orderNumber, orderType, tableNumber, status, items: [{ id, menuItemId, menuItemName, quantity, unitPrice, notes, roundNumber, sentAt, modifiers }] }
    - POST rejects non-cashier/manager with 403 (via requireAuth with "orders","create")
    - GET returns all open orders for session.locationId scoped to session.brandId (for list view — returns lightweight shape: id, orderNumber, tableNumber, orderType, openedAt, itemCount)
  </behavior>
  <action>
    Create src/app/api/orders/route.ts.

    Define zod schemas at top of file:
    - modifierSchema: { modifierOptionId: z.string().uuid().optional(), optionName: z.string(), priceAdjustment: z.number() }
    - itemSchema: { menuItemId: z.string().uuid(), quantity: z.number().int().min(1), unitPrice: z.number().min(0), notes: z.string().optional(), selectedModifiers: z.array(modifierSchema).default([]), addedByUserId: z.string().uuid() }
    - createOrderSchema: { tableNumber: z.number().int().positive().optional(), orderType: z.enum(["table", "counter", "takeaway"]), items: z.array(itemSchema).min(1) }

    POST handler: requireAuth(request, "orders", "create") -> validate body -> db.transaction(tx => compute orderNumber, insert order, insert items with sentAt=new Date(), insert modifiers) -> return 201.

    GET handler: requireAuth(request, "orders", "read") -> query orders where brandId+locationId+status="open" -> return array with item count via subquery or separate count.

    Use Bangkok timezone for daily sequence: format today as YYYY-MM-DD in Asia/Bangkok timezone using `new Date().toLocaleString('en-CA', { timeZone: 'Asia/Bangkok' }).split(',')[0]` then use `sql\`DATE(opened_at AT TIME ZONE 'Asia/Bangkok') = ${today}\``.

    No draft state: all items sent immediately (sentAt = new Date() on insert). roundNumber = 1 for first-round items.

  <behavior>
    - GET /api/orders/:id: returns full order with items grouped by roundNumber. Items sorted: sent (not voided) first grouped by round, then include voidedAt timestamp for voided items. Shape: { id, orderNumber, orderType, tableNumber, status, openedAt, createdByUserId, items: [{ id, menuItemId, menuItemName, quantity, unitPrice, notes, roundNumber, sentAt, voidedAt, voidReason, addedByUserId, modifiers: [{optionName, priceAdjustment}] }] }
    - GET must validate orderId belongs to session.brandId + session.locationId (else 404)
    - PATCH /api/orders/:id: accepts { tableNumber: number } for table transfer. Updates orders.tableNumber. Old table becomes free (no separate update needed — table status is derived from orders). Returns updated order shape.
    - PATCH rejects if order status="completed"
    - POST /api/orders/:id/items: adds new items as a new round. Validates same body as item array from createOrderSchema.items. In transaction: SELECT MAX(round_number) + 1 as nextRound, INSERT items with that roundNumber and sentAt=new Date(). Returns { roundNumber, items: OrderItem[] }
