  <behavior>
    - POST /api/auth/verify-pin: body { pin: string, requiredRole: "manager" | "admin" } → finds all users at session.locationId with requiredRole (or admin) → bcrypt.compare each → returns { valid: true, userId: string } or { valid: false, userId: null }. Does NOT update any session or cookie.
    - POST /api/orders/:id/items/:itemId/void: body { reason: VoidReason, note?: string, authorizedByUserId: string } → validates reason is valid enum value → checks order exists (brandId + locationId scope) → checks item sentAt is not null (if sentAt is null, authorizedByUserId is not required but should still be accepted) → sets voidedAt=new Date(), voidReason, voidNote, voidedByUserId on the order item → returns updated item
    - POST /api/orders/:id/void: body { reason: VoidReason, note?: string, authorizedByUserId: string } → validates → sets voidedAt=new Date() on the order AND all non-voided items in the order → sets orders.status="completed", orders.completedAt=new Date() → returns updated order
    - All void endpoints require requireAuth(request, "orders", "delete") — manager and admin only

    ADD useVoidItem and useVoidOrder to src/hooks/use-order.ts:
    ```typescript
    export function useVoidItem() {
      const queryClient = useQueryClient();
      return useMutation({
        mutationFn: async ({ orderId, itemId, reason, note, authorizedByUserId }) => {
          const res = await fetch(`/api/orders/${orderId}/items/${itemId}/void`, {
            method: 'POST', headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ reason, note, authorizedByUserId }),
          });
          if (!res.ok) throw new Error("Void failed");
          return res.json();
        },
        onSuccess: (_, vars) => {
          queryClient.invalidateQueries({ queryKey: ['order', vars.orderId] });
          queryClient.invalidateQueries({ queryKey: ['tables'] });
        },
      });
    }
    ```
    Similarly for useVoidOrder.

    The VoidReasonDialog should call these hooks directly (import from use-order.ts) rather than raw fetch calls — keeps mutation logic in hooks.
